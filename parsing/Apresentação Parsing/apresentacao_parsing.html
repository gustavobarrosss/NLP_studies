Parte 1

Como os Dados Chegam

O problema antes de qualquer tratamento

1.1 — HTML / Texto com Marcações

O conteúdo chega embrulhado em tags

<h1>CAPÍTULO III</h1>
<h2>Da Política Agrícola e Fundiária</h2>
<p class="artigo"><strong>Art. 184.</strong> Compete à União desapropriar por interesse social...</p>
<!-- Nota: Atualizado pela EC 123 -->
<p class="paragrafo">§ 1º As benfeitorias úteis...</p>
Tags, atributos, CSS, comentários — tudo misturado com o conteúdo real.

1.2 — PDF Nativo (Digital)

Parece texto — mas por dentro é um stream binário

%PDF-1.4
BT /F1 14 Tf 72 770 Td (Art. 184) Tj ET
BT /F1 11 Tf 72 740 Td (Compete a Uniao desapropriar...) Tj ET
...
stream x\x9c\xed]Ko\xe3H\xb2...
Texto misturado com operadores gráficos e metadados de fonte.

1.3 — PDF com Tabelas

O problema central:
Célula (linha=2, coluna=3) não existe no arquivo.
O que existe é: "texto '13.624,02' na posição x=420, y=305"

A tabela é uma ilusão visual — o PDF só tem coordenadas.

1.4 — Documento Escaneado

Pixel[120,45] = (42, 38, 35)   # marrom escuro (tinta?)
Pixel[121,45] = (245, 240, 228) # bege (papel?)
Não há texto — só uma matriz de pixels RGB. Precisa de OCR.

1.5 — Visão Rápida dos Formatos

PDF nativo (texto digital)

PDF escaneado (imagem)

PDF com tabela

Visualmente parecidos para humanos, mas cada formato exige uma estratégia diferente de parsing.

1.6 — Formatos Semi-Estruturados

Nem sempre lidamos com PDFs. APIs, dumps de banco de dados e Diários Oficiais costumam entregar os dados nestes formatos:

JSON

{
  "id": "TRX-123",
  "status": "concluido",
  "tags": ["pix", "urgente"]
}
Padrão web moderno. Estrutura de chave-valor, fácil de parsear (dicionários em Python). O arquivo inteiro precisa ser lido para a memória.

JSONL (JSON Lines)

{"id": 1, "status": "ok"}
{"id": 2, "status": "erro"}
{"id": 3, "status": "ok"}
Um objeto JSON válido por linha. Excelente para streaming e datasets gigantes (NLP/LLMs), pois lê-se linha por linha sem estourar a RAM.

XML

<doc id="123">
  <status>concluido</status>
  <tag>pix</tag>
</doc>
Baseado em tags (irmão mais velho do HTML). Muito comum em sistemas corporativos legados, NF-e e bases de jurisprudência.
O lado bom: A estrutura semântica já existe (não precisa de OCR ou pdfplumber). O desafio aqui é apenas mapear chaves complexas/aninhadas para o seu pipeline.

1.7 — Salvando os Dados Extraídos

Após o parsing, o texto precisa ser persistido. A escolha do formato impacta velocidade, compressão e compatibilidade.

FormatoTipoVantagensLimitaçõesUso típico
.txt Texto puro Universal, legível por qualquer ferramenta Sem estrutura, sem tipos, sem compressão Logs, texto corrido, corpora simples
.csv Tabular (texto) Legível, compatível com Excel e pandas Sem tipos nativos, lento em escala, frágil com texto com vírgulas Dados tabulares pequenos/médios
.parquet Colunar (binário) Compressão alta, leitura colunar rápida, preserva tipos Não legível por humanos, requer biblioteca Pipelines de ML/NLP, Spark em produção, etc
.orc Colunar (binário) Compressão superior ao Parquet em alguns casos, otimizado para Hive/Spark Menor suporte fora do ecossistema Hadoop Data lakes, pipelines Spark em produção
Regra geral: texto corrido → .txt  |  dados tabulares pequenos → .csv  |  pipeline de ML em escala → .parquet

Resumo: O Problema

TipoO que vimosDesafio
HTMLTags, CSS, comentáriosSeparar texto do markup
PDF nativoStream binárioReconstruir ordem do texto
PDF tabelasCoordenadas X,YInferir grade sem metadados
EscaneadoPixels RGBNão há texto — precisa de OCR

A qualidade do seu pipeline de IA começa aqui — antes do embedding, antes do modelo.

Conclusão

Pipeline Híbrido de Produção

Fluxo Completo

Documento chega
  |
  +-- É HTML?
  |    +-- BeautifulSoup → Regex (limpeza) → texto limpo
  |
  +-- É PDF nativo?
  |    +-- Simples → PyMuPDF → Regex
  |    +-- Tabelas → pdfplumber → pandas
  |    +-- Complexo → Docling / Unstructured → Markdown
  |
  +-- É escaneado / imagem?
  |    +-- Tesseract (OCR) → LLM (correção) → texto limpo
  |
  +-- Pós-processamento final
       +-- LLM → JSON estruturado → banco / RAG

Regex aparece em todas as etapas. Nenhuma ferramenta resolve tudo sozinha.
